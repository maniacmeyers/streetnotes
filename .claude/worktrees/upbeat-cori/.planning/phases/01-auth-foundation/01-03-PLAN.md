  <action>
    Create `supabase/migrations/001_initial_schema.sql` with the complete schema from Research.

    The SQL file must contain, in this order:

    1. Enable UUID extension:
    ```sql
    CREATE EXTENSION IF NOT EXISTS "uuid-ossp";
    ```

    2. Create `notes` table:
    - `id` UUID PRIMARY KEY DEFAULT uuid_generate_v4()
    - `user_id` UUID NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE
    - `created_at` TIMESTAMPTZ NOT NULL DEFAULT NOW()
    - `updated_at` TIMESTAMPTZ NOT NULL DEFAULT NOW()
    - `title` TEXT (nullable)
    - `raw_transcript` TEXT (nullable)
    - `structured_output` JSONB (nullable)
    - `status` TEXT NOT NULL DEFAULT 'draft' (values: 'draft', 'reviewed', 'pushed')

    3. Create `crm_connections` table:
    - `id` UUID PRIMARY KEY DEFAULT uuid_generate_v4()
    - `user_id` UUID NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE
    - `crm_type` TEXT NOT NULL ('salesforce' | 'hubspot')
    - `access_token` TEXT (nullable -- will be encrypted in Phase 4)
    - `refresh_token` TEXT (nullable)
    - `token_expires_at` TIMESTAMPTZ (nullable)
    - `created_at` TIMESTAMPTZ NOT NULL DEFAULT NOW()
    - `updated_at` TIMESTAMPTZ NOT NULL DEFAULT NOW()
    - UNIQUE constraint on (user_id, crm_type)

    4. Create `deal_stage_cache` table:
    - `id` UUID PRIMARY KEY DEFAULT uuid_generate_v4()
    - `user_id` UUID NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE
    - `crm_type` TEXT NOT NULL
    - `stages` JSONB NOT NULL DEFAULT '[]'
    - `cached_at` TIMESTAMPTZ NOT NULL DEFAULT NOW()

    5. Enable RLS on all three tables:
    ```sql
    ALTER TABLE notes ENABLE ROW LEVEL SECURITY;
    ALTER TABLE crm_connections ENABLE ROW LEVEL SECURITY;
    ALTER TABLE deal_stage_cache ENABLE ROW LEVEL SECURITY;
    ```

    6. Create RLS policies for each table. CRITICAL: Use `(select auth.uid())` subquery form (not bare `auth.uid()`) for performance -- this lets Postgres cache the result per statement.

    For `notes`: SELECT, INSERT, UPDATE, DELETE policies scoped to authenticated role where user_id matches (select auth.uid()). INSERT and UPDATE use WITH CHECK clause.

    For `crm_connections`: Same four policies (SELECT, INSERT, UPDATE, DELETE).

    For `deal_stage_cache`: Use a single FOR ALL policy since all operations have the same condition.

    7. Create performance indexes on user_id columns:
    ```sql
    CREATE INDEX notes_user_id_idx ON notes (user_id);
    CREATE INDEX crm_connections_user_id_idx ON crm_connections (user_id);
    CREATE INDEX deal_stage_cache_user_id_idx ON deal_stage_cache (user_id);
    ```

    Use the exact SQL from the Research document's Database Schema section. Add a comment header at the top:
    ```sql
    -- StreetNotes.ai Initial Schema
    -- Phase 1: Auth Foundation
    -- Run this in Supabase SQL Editor or via supabase db push
    ```
