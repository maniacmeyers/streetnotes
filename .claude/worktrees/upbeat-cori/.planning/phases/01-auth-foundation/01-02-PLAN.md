  <action>
    Create `app/(auth)/login/actions.ts` with three Server Actions:
    - `login(formData: FormData)`: Calls `supabase.auth.signInWithPassword()` with email and password from formData. On error, redirects to `/login?error=Invalid+credentials`. On success, calls `revalidatePath('/', 'layout')` then redirects to `/dashboard`.
    - `signup(formData: FormData)`: Calls `supabase.auth.signUp()` with email and password. On error, redirects to `/sign-up?error=Could+not+create+account`. On success, `revalidatePath('/', 'layout')` then redirects to `/dashboard`.
    - `signout()`: Calls `supabase.auth.signOut()`, `revalidatePath('/', 'layout')`, redirects to `/login`.
    All three use `createClient()` from `@/lib/supabase/server`. Mark file with `'use server'`.

    Create `app/(auth)/login/page.tsx`:
    - Server Component that renders a login form
    - Form uses the `login` action from `./actions`
    - Email input (type="email", required, name="email")
    - Password input (type="password", required, minLength={6}, name="password")
    - Submit button "Sign in" with min-h-[44px] for mobile tap target
    - Google sign-in button (import `GoogleSignInButton` component)
    - Link to `/sign-up` ("Don't have an account? Sign up")
    - Display error from searchParams.error if present (use a red text alert)
    - All interactive elements: min-h-[44px], min-w-[44px], text-base or larger
    - Mobile-first layout: centered, max-w-sm, px-6, flex flex-col gap-4

    Create `app/(auth)/sign-up/page.tsx`:
    - Similar to login but uses `signup` action
    - Email + password inputs
    - Submit button "Create account"
    - Link to `/login` ("Already have an account? Sign in")
    - Display error from searchParams.error

    Create `app/(auth)/sign-up/actions.ts`:
    - Re-export `signup` from login/actions OR duplicate. Simplest: import and re-export from `../login/actions`.
    - Actually, just put signup in login/actions.ts and import from there. No separate actions file needed -- but for clean file ownership, create this file that imports and re-exports: `export { signup } from '../login/actions'`

    Create `components/google-sign-in-button.tsx`:
    - Client Component (`'use client'`)
    - Uses `createClient()` from `@/lib/supabase/client`
    - onClick calls `supabase.auth.signInWithOAuth({ provider: 'google', options: { redirectTo: \`\${window.location.origin}/auth/callback\` } })`
    - Button styled with min-h-[44px], full width, distinct from primary button (e.g., white bg with border)
    - Text: "Continue with Google"

    Create `app/auth/callback/route.ts`:
    - GET handler that extracts `code` and `next` from searchParams
    - If `next` doesn't start with `/`, set to `/` (prevent open redirect)
    - If code exists, call `supabase.auth.exchangeCodeForSession(code)`
    - On success, redirect to the `next` URL (default `/dashboard`). Handle forwarded host for production (check `x-forwarded-host` header, use HTTPS in production).
    - On failure or no code, redirect to `/auth/auth-code-error`
    - Use exact pattern from Research Pattern 4.

    Create `app/auth/auth-code-error/page.tsx`:
    - Simple page: "Something went wrong during sign in. Please try again."
    - Link back to `/login`
