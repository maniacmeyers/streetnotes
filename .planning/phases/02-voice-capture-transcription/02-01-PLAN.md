  <files>
    lib/audio/recording.ts

  <action>
    Create `lib/audio/recording.ts`:
    - Define `PREFERRED_AUDIO_MIME_TYPES` in priority order:
      1. `audio/webm;codecs=opus`
      2. `audio/mp4`
      3. `audio/ogg;codecs=opus`
    - Export `MAX_AUDIO_BYTES = 25 * 1024 * 1024`
    - Export `pickSupportedMimeType()` with MediaRecorder feature detection
    - Export `extensionForMimeType()` to map MIME to filename extension

    Create `hooks/use-voice-recorder.ts` as a Client hook:
    - State: `status` (`idle | requesting_permission | recording | stopped | error`), `durationSec`, `audioBlob`, `mimeType`, `error`
    - Methods: `startRecording()`, `stopRecording()`, `resetRecording()`
    - Internals:
      - Request microphone stream with `navigator.mediaDevices.getUserMedia({ audio: true })`
      - Instantiate `MediaRecorder` with negotiated MIME (if available)
      - Collect chunks from `ondataavailable`
      - Build Blob in `onstop`
      - Track duration via `setInterval` while recording
      - Ensure stream tracks are stopped after stop/reset
      - Gracefully handle permission denial / unsupported browser
  </action>
  <verify>
    Run:
    - `npm run lint`
    - `npm run build`
    Confirm TypeScript signatures:
    - `useVoiceRecorder` returns typed state + methods
    - `pickSupportedMimeType()` never throws when unsupported
  </verify>
  <done>
    Recorder hook exists and provides a stable recording contract for UI and upload flows.
  </done>
